# TJ-Social 环境健康分设计规格

> 设计主理：Nina · 版本 v1.0 · 2026-07-08
> 设计参考：linear.app（超紧密深色产品画布）+ supabase（数据密集表格视图）

---

## 1. 设计语言衔接

### Token 映射关系

小白兔 TKNC 的 tokens.css 是本方案的基础，以下是主平台 TJ-Social 后台的语义映射：

| tokens.css 变量 | 语义角色 | 主平台用途 |
|---|---|---|
| `--bg` `#0a0e1a` | 画布底色 | 侧边栏/全局背景 |
| `--bg-elevated` `#111827` | 一级抬升面 | 内容区底板 |
| `--panel` `#1a2234` | 二级面板 | 卡片/表格行悬停 |
| `--panel-2` `#232c42` | 三级面板 | 选中行/焦点区域 |
| `--border` `rgba(255,255,255,0.08)` | 分隔线 | 表格行线/卡片边框 |
| `--brand` `#00d4ff` | 品牌蓝 | 主要交互/进度环 |
| `--tier-excellent` `#10b981` | 优秀绿 | ≥90 分/放行状态 |
| `--tier-good` `#facc15` | 良好黄 | 70-89 分/二次确认 |
| `--tier-warning` `#f97316` | 警告橙 | 50-69 分/需关注 |
| `--tier-danger` `#ef4444` | 危险红 | 0-49 分/拦截状态 |
| `--text` `#e6ebf5` | 主文字 | 标题/账号名 |
| `--text-muted` `#8b95ab` | 次要文字 | 元信息/时间戳 |

**从 linear.app 借鉴的设计决策：**
- 用面板颜色梯度（`--bg` → `--panel-2`）做层级，不用阴影
- 发布前弹窗用 `--border-strong` 描边 + `--panel` 底色，避免 shadow 堆叠
- 表格行 1px `--border` 分隔，悬停升至 `--panel`，不用颜色改变

**从 supabase 借鉴的设计决策：**
- 数据表格：紧凑行高 40px，对齐左侧文字基线
- 分数和 tier badge 做成行内 pill，不另起一列
- 撞车预警视图：热力格（heatmap grid），类似 supabase 的日志频率图

## 2. 分数/Tier/Severity 视觉映射

### Tier 颜色规则（对应 scoring.js tierOf()）

```
excellent ≥90  →  --tier-excellent  #10b981  绿色  "优秀"
good      ≥70  →  --tier-good       #facc15  黄色  "良好"
warning   ≥50  →  --tier-warning    #f97316  橙色  "警告"
danger    <50  →  --tier-danger     #ef4444  红色  "危险"
```

### Severity 徽章规则（对应 recommendations.js severity 字段）

| severity | 背景色（半透明） | 文字色 | 图标 | 用途 |
|---|---|---|---|---|
| `danger` | `rgba(239,68,68,0.15)` | `#ef4444` | 八角警告 ⛔ | 强阻断问题 |
| `warn` | `rgba(249,115,22,0.15)` | `#f97316` | 三角 ⚠ | 需关注问题 |
| `info` | `rgba(59,130,246,0.15)` | `#3b82f6` | 圆圈 ℹ | 提示信息 |

### 分数圆环（行内版）

后台不用完整大圆环，改用 **24px 小圆圈数字 + 分数文字** 的行内格式：

```
[●92] 优秀     ← 实心彩圆 + 分数（一行压缩）
```

大分数：账号详情页用 60px 圆环，取自现有 score-circle 样式

## 3. 场景一：发布前预检

### 信息架构

用户在 TJ-Social 点"发布"→ 系统在后台静默跑预检（或弹出面板）→ 依 tier 结果决定是否放行。

**三种拦截态：**

| Tier | 交互行为 | 视觉样式 |
|---|---|---|
| excellent / good | 静默放行，左下角 toast 提示 | 绿/黄小 toast，2s 自动消失 |
| warning (50-69) | 弹出二次确认 Dialog | 橙色标题区 + "仍要发布"确认按钮 |
| danger (<50) | 阻断发布，强制 Dialog | 红色标题区 + 禁用发布按钮 |

### 预检 Dialog 布局（warning/danger）

```
┌─────────────────────────────────────────────┐  --panel 底色
│  ⚠ 环境预检警告                      [×]   │  标题栏 --panel-2
│─────────────────────────────────────────────│
│  分数环 [68]    tier: 警告                  │
│                                             │
│  关键问题（前3条 danger/warn）：            │
│  ┌─────────────────────────────────────┐   │
│  │ ⛔ IDC 机房 IP 检测到               │   │  danger 卡片
│  │  强烈建议切换住宅代理后再发布        │   │
│  └─────────────────────────────────────┘   │
│  ┌─────────────────────────────────────┐   │
│  │ ⚠  时区与 IP 国家不匹配            │   │  warn 卡片
│  └─────────────────────────────────────┘   │
│                                             │
│  [查看完整报告]         [仍要发布 / 已阻止] │
└─────────────────────────────────────────────┘
```

### Dialog 组件规格

- **shadcn 组件**：`Dialog` + `DialogTitle`（必填，可 sr-only）+ `DialogDescription`
- 标题栏颜色：warning → `rgba(249,115,22,0.1)` 背景；danger → `rgba(239,68,68,0.1)` 背景
- 建议卡片：`--panel-2` 底色，`--r-md` 圆角，`--sp-4` 内边距
- "仍要发布"按钮（warning）：`--tier-warning` 背景，`--bg` 文字，hover 降低 opacity
- "已阻止发布"按钮（danger）：disabled 态，`--panel` 底色，`--text-dim` 文字
- "查看完整报告"按钮：ghost 态，`--text-muted` 文字

### 预检运行中状态

弹出一个轻量 Sheet（从底部/侧边滑入），展示实时进度：

- 6 个模块图标依次打勾
- 进度条用 `--brand` 色
- 完成后自动切换到结果视图（不关闭，原地替换内容）

### 静默放行 Toast

- **shadcn 组件**：`Sonner` toast（标准位置，右下角，不做弹窗）
- excellent：绿色左边框 + "✓ 环境良好（92分），已放行发布"
- good：黄色左边框 + "⚠ 环境一般（76分），已放行，建议关注"

## 4. 场景二：账号健康分（列表 + 详情）

### 账号列表列设计

主账号矩阵表格新增"健康分"列，位置建议在账号名后第二列：

```
列顺序：
  账号名 | 健康分 | 上次检测 | 状态 | 操作
```

**健康分列的行内呈现：**

```
[●92 优秀]          ← tier pill + 分数
[●74 良好]
[●61 ⚠ 警告]       ← warning 加三角角标
[●32 ✕ 危险] ↓5   ← danger + 趋势箭头
```

- **Pill 规格**：`--r-pill` 圆角，`--sp-2` + `--sp-3` 内边距（垂直/水平），12px 字号
- **趋势箭头**：分数较上次降低时显示 `↓` + 变化值（`--tier-danger` 色）；上升显示 `↑`（`--tier-excellent` 色）
- **预警角标**：danger 账号的账号名前显示红点 `●`，列表会按 tier 排序（危险在前）

### 健康分列表行的完整信息密度（参考 supabase 表格）

```
┌────┬────────────────┬──────────────┬──────────┬────────────────┬─────────┐
│ ☐  │ 账号名          │ 健康分        │ 上次检测 │ 状态           │ 操作    │
├────┼────────────────┼──────────────┼──────────┼────────────────┼─────────┤
│ ☐  │ ● @us_account1 │ ●92 优秀     │ 2h 前   │ ○ 活跃         │ 检测 …  │
│ ☐  │ @us_account2   │ ●74 良好 ↑3  │ 1d 前   │ ○ 活跃         │ 检测 …  │
│ ☐  │ ⚠ @jp_account1 │ ●61 警告 ↓8 │ 3d 前   │ ◎ 待观察       │ 检测 …  │
│ ☐  │ ✕ @cn_risky    │ ●32 危险 ↓22│ 5d 前   │ ✕ 暂停         │ 检测 …  │
└────┴────────────────┴──────────────┴──────────┴────────────────┴─────────┘
```

- **行高**：40px（紧凑，参考 supabase 日志流行高）
- **悬停**：行背景升至 `--panel`
- **选中**：`--panel-2` 底色 + 左侧 `--brand` 色 2px 边框
- **shadcn 组件**：`Table` + `TableRow` + `Badge` variant（自定义 tier 颜色）

### 单账号详情页

```
┌─ 账号详情 ────────────────────────────────────────────────────────────┐
│  头像  @us_account1  [●92 优秀]  上次检测: 2025-07-07 14:32           │
│  [立即重新检测]  [查看历史]                                           │
├────────────────────────────────────────────────────────────────────────┤
│  检测报告概览                                                          │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐  │
│  │ IP身份 │ │ DNS    │ │WebRTC  │ │稳定性  │ │设备    │ │可达性  │  │
│  │  85/25%│ │  90/15%│ │100/15% │ │  78/25%│ │100/10% │ │100/10% │  │
│  │ ●良好  │ │ ●优秀  │ │ ●优秀  │ │ ●良好  │ │ ●优秀  │ │ ●优秀  │  │
│  └────────┘ └────────┘ └────────┘ └────────┘ └────────┘ └────────┘  │
├────────────────────────────────────────────────────────────────────────┤
│  改进建议                                    健康分趋势（30天）        │
│  ⚠ IP 被标记为代理                          92 ─────────── 87─91─92  │
│  ⚠ 延迟偏高（420ms）                                                  │
│  ℹ 完整 DNS 检测需付费版                                               │
└────────────────────────────────────────────────────────────────────────┘
```

**6 项模块卡片规格：**
- 每个卡片：`--panel` 底色，`--r-md` 圆角，内含分数 + 权重 + tier pill
- 排列：PC 6列一行，响应式 3×2
- 点击可展开模块细节（Collapsible）

**改进建议规格：**
- 用 severity 卡片（同场景一建议卡片样式）
- danger 排前，warn 次之，info 最后
- **shadcn 组件**：`Accordion` 展开详细说明

**健康分趋势：**
- 简单折线（SVG 或 recharts），30天区间
- 横轴：日期；纵轴：0-100；tier 颜色填充区域
- 如无历史数据，显示 empty state：`--panel` 底，`--text-dim` 提示"暂无历史，检测后自动记录"

## 5. 场景三：矩阵撞车预警

### 核心问题

批量养号最大风险：多个账号共用同一个 IP 或相似指纹 → TikTok 关联检测 → 集体限流。

### 信息架构

撞车预警是一个独立的视图/页签，入口放在账号矩阵页顶部导航 Tab 中：

```
账号矩阵  |  健康监控  |  [撞车预警 ⚠3]  |  发布计划
```

角标数字 = 当前有撞车问题的 IP 冲突组数量。

### 撞车预警视图布局

```
┌─ 撞车预警 ─────────────────────────────────────────────────────────────┐
│  [本次检测: 42个账号]  [⚠ 发现3组IP冲突]  [重新扫描]                  │
├──────────────────────────────────────────────────────────────────────────┤
│  IP 冲突热力图                                                           │
│                                                                          │
│  IP 地址          账号 →  @acc1  @acc2  @acc3  @acc4  @acc5  …         │
│  104.21.xx.xx   ──────   [🟥]   [   ]  [🟥]   [   ]  [🟥]             │
│  172.67.xx.xx   ──────   [   ]  [🟨]   [🟨]   [   ]  [   ]             │
│  23.227.xx.xx   ──────   [   ]  [   ]  [   ]  [🟩]   [   ]             │
│  …                                                                        │
│                                                                           │
│  冲突说明：🟥 = 3+账号共用  🟨 = 2账号共用  🟩 = 独享                  │
├──────────────────────────────────────────────────────────────────────────┤
│  冲突组详情                                                              │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │ ⛔ 高风险：IP 104.21.xx.xx 被 3 个账号共用                      │   │
│  │ 涉及账号：@us_acc1  @jp_acc3  @us_acc7                          │   │
│  │ [立即隔离这3个账号的代理]                                         │   │
│  └──────────────────────────────────────────────────────────────────┘   │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │ ⚠ 中风险：IP 172.67.xx.xx 被 2 个账号共用                       │   │
│  │ 涉及账号：@kr_acc2  @kr_acc4                                     │   │
│  └──────────────────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────────────┘
```

### 热力格（Heatmap Cell）规格

| 状态 | 含义 | 颜色 |
|---|---|---|
| 红格 🟥 | 该账号使用此 IP，且已有 2+ 个账号共用 | `rgba(239,68,68,0.6)` + `--tier-danger` 描边 |
| 黄格 🟨 | 该账号使用此 IP，与 1 个账号共用 | `rgba(249,115,22,0.4)` + `--tier-warning` 描边 |
| 绿格 🟩 | 该账号使用此 IP，独享 | `rgba(16,185,129,0.2)` + `--tier-excellent` 描边 |
| 空格 | 该账号不使用此 IP | `--panel` 底色无描边 |

- **单格尺寸**：28×28px，间距 2px
- **账号列表**：水平滚动，固定左侧 IP 列（sticky）
- **hover**：弹出 Tooltip 显示"账号名 使用 IP x.x.x.x，与 N 个账号共用"

### 指纹撞车检测（扩展项）

除 IP 外，也可检测 User-Agent、时区、语言组合的相似度。当多个账号"指纹"相同时，在热力图旁增加"指纹冲突"列：

```
IP 冲突  |  指纹冲突  （两个子 Tab）
```

指纹冲突同样用格子图，颜色映射相同。

### shadcn 组件

- `Tabs` — IP冲突/指纹冲突切换
- `Tooltip` — 格子 hover 信息
- `Badge` — 账号 tier 标注
- `ScrollArea` — 热力图水平滚动

## 6. 通用交互状态清单

### 加载态

- 表格行：用 `Skeleton` 组件，`--panel` 底色占位块（参考 shadcn skeleton）
- 分数圆环：旋转 indeterminate 动画（复用 tokens.css `--ease-out` + `--dur-slow`）
- 热力图：全格灰色 `--panel` 占位，等待 API

### 失败态

- 某个模块检测失败：该模块卡片加 `?` 图标 + `--text-muted` "检测失败"，分数显示 0
- 整体网络不通：全屏 empty state，`--tier-danger` 图标 + "网络连接失败，请检查代理"
- **shadcn 组件**：`Alert` variant="destructive" 展示失败摘要

### 空状态

- 账号列表无账号：`--panel` 底色卡片，`--text-dim` 提示"还没有绑定账号"
- 撞车预警无冲突：绿色 empty state，`--tier-excellent` 图标 + "当前无撞车风险，各账号 IP 独立"

## 7. 组件复用与新增建议

### 现有 tokens.css 可直接复用

所有颜色 token、间距 token、动画 token 均可直接引用，无需重新定义。

### 需新增的 shadcn 组件

| 组件 | 用途 | 是否已有 |
|---|---|---|
| `Table` + `TableRow` | 账号列表 | 待添加 |
| `Badge` with tier variants | 健康分 pill | 待添加 |
| `Dialog` | 预检拦截弹窗 | 已有 |
| `Tabs` | 撞车预警切换 | 待添加 |
| `Tooltip` | 热力格说明 | 待添加 |
| `Skeleton` | 加载占位 | 待添加 |
| `Accordion` | 建议展开 | 待添加 |
| `ScrollArea` | 热力图滚动 | 待添加 |
| `Alert` | 失败态摘要 | 待添加 |

### 需沉淀为可复用组件的手搓件

1. **`<HealthBadge tier score>`** — tier pill（出现在列表行、详情标题、预检弹窗）
2. **`<ModuleCard name score weight tier>`** — 6项模块卡片
3. **`<SeverityCard severity title body>`** — 建议卡片（同时用于预检弹窗和详情页）
4. **`<HeatCell status>`** — 热力格单元

以上四个组件建议放在 `components/health/` 目录，内部使用 `cn()` + `cva` 管理 tier/severity variant。
